基于知识图谱的教学资源管理系统 - 开发交接文档
一、项目基础信息
1. 项目目标
开发实现教学资源结构化存储、语义检索、图谱路径检索、可视化图谱导航、个性化推荐的教学资源管理系统，基于知识图谱实现多维度资源管理与智能检索推荐。
2. 技术栈
后端 : Python 3.13+ + Flask 框架 
数据库 : MySQL（用户/RBAC/资源元数据）、Neo4j（知识图谱实体与关系）
前端 : Vue 3 + Vite + Element Plus 
图谱可视化 : ECharts（已落地）/ G6 

二、标准开发流程（5个阶段）
阶段1：数据库设计补全（从“能用”到“可扩展”）
1. MySQL：补齐资源元数据表字段（审核人/时间、分类体系、版本、下载次数等），RBAC权限细化到“权限码”；
2. Neo4j：确定节点/关系字段规范（课程/知识点/资源/教师唯一键、属性、关系类型），明确资源与知识点绑定方式。

阶段2：后端API完整化（落地全业务流程）
1. 资源模块：实现上传→待审→审核通过/拒绝→下载权限控制，补充分页、排序、删除/更新、资源详情、上传校验与文件存储策略；
2. 用户模块：实现管理员创号/改角色、token刷新、密码重置，基于RBAC实现三级角色权限；
3. 图谱构建：开发MySQL与Neo4j的批量导入/同步脚本/接口，将模拟实体识别结果写入图谱。

阶段3：前端完善与联调（实现可交互功能）
1. 资源模块：实现列表分页/筛选/详情、管理员审核页、上传进度/状态提示；
2. 个人中心：实现学习轨迹、收藏夹、推荐列表与理由展示；
3. 图谱页：实现节点点击联动查询、搜索框、按课程/章节过滤，完成图谱可视化交互。

阶段4：智能检索与推荐（实现智能核心功能）
1. 语义检索：基于Neo4j路径查询实现“关键词+关联知识点”检索，将结果解释返回前端；
2. 图谱路径检索：用户在前端可视化图谱上点击某个节点（比如点击「Python 函数」知识点），系统会通过路径检索，自动加载该节点的所有关联节点 + 资源（比如函数知识点的先修知识点、关联课件、上传教师），实现图谱的交互式浏览；
3. 推荐增强：基于“用户浏览/收藏行为+图谱邻接关系”实现可解释推荐，将图谱路径与用户行为权重设为可调参数，优化推荐精准度。

阶段5：系统化（上线前标准化落地）
1. 联调：统一CORS/鉴权错误码，解决前后端接口跨域/权限问题；
2. 运维功能：开发MySQL/Neo4j数据备份接口，实现系统日志与审计、接口性能限流、关键接口慢查询/失败率监控及异常告警。

四、项目关键入口与文件
后端
1. 主入口：`backend/app.py`
2. 依赖：`backend/requirements.txt`
3. 配置：根目录`.env`
4. 核心命令：
   - 数据库初始化：`python backend/app.py init-db`
   - 生成测试数据：`python backend/app.py seed-demo`
   - 图谱同步：`python backend/app.py sync-neo4j --reset`

前端
1. API配置：`frontend/.env.development`
2. 路由：`frontend/src/router/index.js`
3. 核心组件：
   - 图谱页：`GraphView.vue`
   - 资源列表/详情：`ResourceListView.vue`/`ResourceDetailView.vue`
   - 管理员页：`AdminAuditView.vue`/`AdminLogsView.vue`
   - 个人中心：`ProfileView.vue`

五、新账号开发前置操作
1. 拉取/拷贝项目文件，修改`.env`适配新机器的MySQL/Neo4j配置；
2. 激活conda环境`teaching_pro`，安装`backend/requirements.txt`依赖；
3. 依次执行`init-db`/`seed-demo`/`sync-neo4j --reset`，启动前后端即可开发。
